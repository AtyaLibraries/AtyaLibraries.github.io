Simple rules, safer reuse.

Each package is built to remain understandable in isolation and dependable in larger systems.

  • Low-level packages never depend on high-level hosts or integrations.
  • Vendor-specific behavior lives in dedicated adapter packages.
  • Packages can be adopted individually rather than as an all-or-nothing framework.
  • Tests, README documentation, and package metadata are part of the release standard.
